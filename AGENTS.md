# Project Philosophy: The Alabaster Spec Sheet

This project is a "Technical Authority" showcase for Light Yagami, a **Web Systems Developer**. It moves away from generic "full-stack" portfolios toward an industrial, high-spec engineering aesthetic.

> **Note:** For visual design, color tokens, typography rules, and motion guidelines, refer to [STYLE-GIDE.md](./STYLE-GIDE.md).

## Core Identity & Tone
- **Persona:** Senior Web Systems Developer focusing on "hard paths" (Rust, Wasm, custom CI/CD, System Architecture).
- **Tone:** Witty, unfiltered, and confident. Avoid corporate fluff and "entry-level" tropes (no progress bars, no skill percentages). If it sounds like a LinkedIn bio, rewrite it. It should sound like a technical manual.
- **Value:** "Respect for the Craft" — prioritize technical depth and architectural logic over generic application building.

## Architectural Mandates
- **Performance:** Use Next.js PPR (Partial Prerendering) for high-performance reactivity.
- **Diagrams > Screenshots:** Prioritize SVG-based architectural diagrams and system post-mortems over standard UI screenshots.
- **Domain Authority:** Categorize expertise into "Core Systems," "Infrastructure," and "Architecture" instead of a flat list of skills.

## Strict Constraints (The No-Fly Zone)
- **Zero Fluff:** No "passionate about coding" or "innovative solutions."

## Technical Documentation Framework
- **The "Why" over the "How":** When discussing Quix or ThriveDesk, the agent must emphasize architectural trade-offs (e.g., "Why Redux was chosen over Context for X constraint") rather than just listing features.
